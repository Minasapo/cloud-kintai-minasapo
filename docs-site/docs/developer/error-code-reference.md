# エラーコード一覧

このページは `src/errors.ts` で管理するメッセージコードの開発者向けリファレンスです。UI や通知文言で共通利用するメッセージは原則としてここを正本にし、呼び出し側では `@/errors` を参照します。

## 利用方針

- 画面や hooks でメッセージ文字列を直書きせず、`@/errors` の export を利用する。
- 既存コードを変更する場合は、メッセージ値だけでなくカテゴリと用途も確認する。
- 新しいコードを追加する場合は `src/errors.ts` のカテゴリ定義と、このドキュメントの一覧を同時に更新する。
- 既存呼び出し互換のため、`E01001` のような named export は維持する。

## 特記事項

- `E02004`: 同一スタッフ・同一日付の勤怠レコード重複を表す。DB 上の重複解消が必要。
- `E05006`: Cognito からユーザー情報を取得できないケースを表す。
- `E05008`: Cognito のユーザーロール未設定を表す。
- `E05009`: Cognito の `custom:owner` 未設定または不正値を表す。

## 実装メモ

- 実装側では `MESSAGE_CATEGORY_ENTRIES` / `MESSAGE_DEFINITIONS` / `getMessageDefinition()` を利用すると、カテゴリや補足情報まで参照できます。
- 既存画面は引き続き `E01001` のような named export をそのまま利用できます。
