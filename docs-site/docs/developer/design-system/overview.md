# デザインシステム概要

このセクションは、画面ごとの見た目のぶれを抑え、既存 UI と自然につながる実装判断を揃えるためのガイドです。

## 目的

- 実装のたびに色、余白、角丸、情報密度がぶれない状態を維持する
- 新規画面でも既存ルールから選んで組み立てられるようにする
- レビュー時に「雰囲気」ではなく、具体的なルールで差分を確認できるようにする

## 基本方針

- 新しい見た目を都度発明せず、既存 token と primitive を優先する
- 画面単位の個性より、システム全体の一貫性を優先する
- モバイル基準で密度を決め、デスクトップでは読みやすさと最大幅を調整する
- 一時的な feature 固有スタイルを増やす前に、shared へ戻せるかを確認する

## 適用範囲

このガイドは、本プロジェクトの画面実装と UI レビューを対象にします。

- 対象: レイアウト、見た目、画面構成、UI コンポーネントの拡張判断
- 対象外: 文言設計、業務仕様そのもの、ブランド戦略の再定義

## Source Of Truth

デザインルールの正本は、この開発者向けガイドです。

- 旧 `docs/AI_DESIGN_SYSTEM_GUIDE.md` は移行案内としてのみ扱います
- 補助資料や一時メモは `docs/` に置けますが、恒久的なルールはこのサイトを更新します

## 実装の参照先

実装と接続するときは、まず次を確認してください。

- `src/shared/designSystem/` — token 定義と CSS variable 化
- `src/shared/ui/layout/PageSection.tsx` — ページ内の基本セクションレイアウト
- `/preview/design-tokens` — token の見た目を確認するための社内プレビュー画面
