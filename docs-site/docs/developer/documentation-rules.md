# ドキュメント運用ルール

このページは、ドキュメントサイト全体の情報設計と運用ルールをまとめたガイドです。

利用者向け、開発者向けを問わず、恒久的に参照する仕様や手順はドキュメントサイトを正本として管理します。リポジトリ直下の `docs/` には、リポジトリ運用や公開手順のようにサイトへ載せるべきでない資料、または移行途中の補助メモだけを残します。

## 基本方針

- 読み手が継続的に参照する内容はドキュメントサイトに置く
- 共通概念、ロール別操作、開発者向け仕様を混在させず、入口を分ける
- サイドバー順、overview ページ、関連リンクをセットで更新し、孤立ページを作らない
- 用語は [用語集](../terminology) を基準に揃える
- 共通ガイドは `業務フロー全体像` → `用語集` → `勤務ステータス` → `勤務形態` の順で参照する

## このサイトの分類基準

### 共通ガイド

複数ロールで共有する概念、索引、業務全体像を置きます。

- `intro`
- `attendance-workflow-overview`
- `terminology`
- `work-status-overview`
- `work-type-overview`
- `screen-list`

### スタッフ向け / 管理者向け

実際の業務操作や画面利用手順をロール別に置きます。`overview` を入口にし、その配下を `navigation`、`use-cases`、`features`、`faq` で揃えます。

- やりたいことから辿るものは `use-cases`
- 画面名、機能名から辿るものは `features`
- 日々の具体的な操作手順は各ロール配下の個別ページ
- `overview` は「共通前提」「目的別」「機能別」「困ったとき」の順で入口を案内する

### 開発者向け

開発参加者が参照するセットアップ、設計資料、実装仕様、運用ルールを置きます。

- `overview` は新規参加者向けの読む順を案内する
- `reference-index` は設計資料、仕様、実装ルールを逆引きする
- 導入と日常開発
- Platform / Amplify
- アーキテクチャ
- デザインシステム
- 機能仕様

### 特殊運用

`/office*` のようなオフィスモード画面は独立ロールとして扱わず、管理者設定に紐づく特殊運用として分類します。画面索引では `特殊運用` セクションに置き、正本説明は管理者向け設定資料に寄せます。

## 追加先の判断基準

### ドキュメントサイトに置くもの

- スタッフまたは管理者が使う画面の操作手順
- 複数ロールにまたがる業務フローや概念整理
- 開発者が継続的に参照する設計、仕様、実装手順
- 今後も更新され続ける正本ドキュメント

### リポジトリ直下の `docs/` に残すもの

- OSS 同期、公開作業、機密情報除去などのリポジトリ運用資料
- 一時的な調査メモや移行途中の補助資料
- 利用者向けまたは通常開発フロー向けではない内部限定メモ

## 命名ルール

- 操作ガイドは `〜する` で統一する
- 概念整理や索引は `〜の見方`、`〜全体像`、`〜一覧` を使う
- 実装寄りの資料は `〜仕様`、`〜手順`、`〜ルール` を使う
- `overview` は「誰向けの入口か」と「どう辿るか」を最初に書く
- `use-cases` は「目的から探す入口」、`features` は「画面名・機能名から探す入口」と明記する
- `description` は 1 文で対象と目的を言い切る
- 判断や運用が必要な節は、箇条書きの前に短い説明段落を置き、項目の意味や読む場面を先に示す

## 更新ルール

1. 新規ページ追加時は `sidebars.ts` に反映する
2. 入口ページから辿るべき内容なら、該当する `overview` も更新する
3. ロール別ページなら `use-cases` と `features` のどちらから辿るべきかを決める
4. 画面索引に影響する変更なら `screen-list` も更新する
5. 設定画面や特殊運用画面を追加した場合は、`settings-item-list` の棚卸しだけでなく個別の運用ガイドも追加する
6. 共通ページへ寄せられる説明は重複記述せず、リンクで参照する
7. 変更後は `npm --prefix docs-site run build` で検証する

## 既存の外部 docs の扱い

### `docs/SCREEN_LIST_BY_ROLE.md`

このファイルは画面とルートを粗く棚卸しするための補助資料です。利用者や開発者が参照する正本の画面索引は [画面一覧](../screen-list) とし、新しい画面追加時はまずこのページを更新します。

この補助資料を残す場合でも、ドキュメントサイトと二重管理しないことを優先し、内部棚卸しに必要な粒度だけを保持します。

### `docs/OSS_SYNC_GUIDE.md`

このファイルは OSS リポジトリへの同期手順を扱うリポジトリ運用資料です。利用者向けガイドや通常の開発者向け仕様とは性質が異なるため、ドキュメントサイトへは移しません。

## 変更時チェックリスト

- ページの置き場所が分類基準に合っている
- タイトルと `description` が役割を表している
- `sidebars.ts` と入口ページから辿れる
- 共通ガイドの参照順が崩れていない
- `use-cases` と `features` のどちらを主導線にするか明確
- 特殊運用を独立ロールとして扱っていない
- 設定一覧ページだけで完結させず、必要な運用ページが用意されている
- 類似説明を既存ページと重複させていない
- `npm --prefix docs-site run build` が通る
