# サイドバーカテゴリ運用ルール

このページは、ドキュメント数が増えてもサイドバーの可読性を維持するための運用ルールをまとめたガイドです。

## 基本方針

- 並び順の管理は `sidebars.ts` に集約する。
- `frontmatter` は `title` と `description` などメタ情報に限定する。
- 1カテゴリに項目が増えすぎる場合は、責務単位で下位カテゴリへ分割する。

## カテゴリ分割の目安

- 1カテゴリの直下項目が 7 件を超えたら分割を検討する。
- 役割が異なる項目が混在したら分割する。
- 利用者が次に開くページを予測しづらくなったら分割する。

## 更新ルール

1. 新規ページ追加時は `sidebars.ts` へ必ず追記する。
2. `docs/developer/overview.md` など入口ページからのリンクも同時に更新する。
3. 既存カテゴリに追加するか、新規カテゴリを作るかをレビューで明示する。

## 変更時チェックリスト

- サイドバー上の順序が利用フローと一致している。
- 同名カテゴリや意味が重複するカテゴリがない。
- overview ページからサイドバー項目へ導線が通っている。
- `npm --prefix docs-site run build` でリンク切れがない。
