このページは、このドキュメントサイト全体の構造を把握するためのサイトマップです。トップページは役割ごとの入口を選ぶランディングとして使い、このページでは「どの種類の情報がどこにあるか」を説明します。

## まず読む共通ガイド

共通理解をそろえるときは、次の順で確認します。

2. [用語集](./terminology)
3. [勤務ステータスの見方](./work-status-overview)
4. [勤務形態の見方](./work-type-overview)

## ロール別ガイドの入口

- スタッフ向け: [スタッフ向けガイド](./staff/overview)
- 管理者向け: [管理者向けガイド](./admin/overview)
- 開発者向け: [開発者向けガイド](./developer/overview)

ロール別ガイドの中では、`overview` を入口にして、`navigation`、`use-cases`、`features`、`faq` の順に辿れる構造にしています。

## 各ページの役割

- `overview`: そのロールや読者向けの入口ページ
- `navigation-map`: 主要な画面遷移や導線の俯瞰
- `use-cases`: やりたいこと、業務目的から探す入口
- `features`: 画面名や機能名から探す入口
- `screen-list`: 画面名、URL、関連ガイドを逆引きする索引
- `faq`: 例外対応やよくある迷いどころの整理

## このサイトの構成

- 共通ガイド: 業務全体像、用語、表示ルール、画面索引
- スタッフ向け: 日々の打刻、確認、申請、個人設定の導線
- 管理者向け: 承認、勤怠確認、シフト、設定管理の導線
- 開発者向け: 導入フロー、逆引きリファレンス、実装ルール

利用者向け、開発者向けの恒久的な仕様や手順は、このドキュメントサイトを正本として管理します。リポジトリ直下の `docs/` 配下には、OSS 同期のようなリポジトリ運用資料や移行途中の補助メモだけを残します。
