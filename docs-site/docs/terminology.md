# 用語集

このページは、ドキュメント内で利用する主要用語を定義するための一覧です。

| 用語 | 簡単な説明 |
| --- | --- |

| [勤務形態](#term-work-type) | スタッフごとの働き方の種類です。 |
| [現在の勤務ステータス](#term-current-work-status) | 打刻画面で使う現在進行中の勤務状態です。 |

| [特別休暇](#term-special-leave) | 有給休暇とは別に付与される休暇です。 |

以下で各用語を詳しく説明します。

## 勤怠 {#term-attendance}

スタッフの勤務時間、出勤・退勤、休憩などの勤務実績全体を指します。給与計算や法令順守の基礎データになります。

## 勤務ステータス {#term-work-status}

本ドキュメントでは、勤務に関する状態表示の総称として使います。ただし実際には「現在の勤務ステータス」と「勤怠判定ステータス」の 2 種類があります。

## 勤務形態 {#term-work-type}

スタッフごとに設定される働き方の種類です。本アプリでは `平日勤務` と `シフト勤務` の 2 種類を扱います。勤務ステータスとは別の概念であり、休日の扱いやシフト運用に影響します。詳細は [勤務形態の見方](./work-type-overview) を参照してください。

## 現在の勤務ステータス {#term-current-work-status}

打刻画面で表示される、現在進行中の勤務状態です。出勤前、勤務中、休憩中、勤務終了などが該当し、その時点で押せる打刻操作を判断するために使います。

## 勤怠判定ステータス {#term-attendance-judgment-status}

勤怠一覧や管理画面で表示される、その日の記録結果に対する判定です。OK、要確認、申請中、表示なしなどが該当し、確認や対応の優先度判断に使います。

## 直行・直帰モード {#term-direct-travel-mode}

現場への移動方法を打刻時に記録するためのモードです。勤務ステータスや勤怠判定ステータスとは別の概念であり、状態名そのものではありません。

- 直行: 所属拠点や事務所を経由せず、現場へ直接向かう出勤打刻
- 直帰: 現場から所属拠点や事務所へ戻らず、直接帰る退勤打刻
- 直行・直帰: 上記の直行と直帰を総称した打刻モード

直行・直帰を利用しても、現在の勤務ステータスや勤怠判定ステータスの考え方自体は変わりません。勤怠レコード上では `goDirectlyFlag` と `returnDirectlyFlag` として記録され、管理画面の一覧や CSV 出力で確認できます。

また、勤怠システム上は、直行は打刻ボタンを押した時刻そのものではなく AppConfig で定義された勤務開始時間で記録され、直帰は AppConfig で定義された勤務終了時間で記録されます。

## 打刻エラー {#term-time-record-error}

出勤・退勤・休憩のいずれかの打刻が不足または不整合な状態を指します。スタッフが修正申請し、管理者が承認することで解消します。

## 要確認 {#term-needs-confirmation}

勤怠判定ステータスの 1 つで、出勤・退勤・休憩の不足や不整合があり、内容確認または修正が必要な状態です。内部的には `Error` に対応します。

## 修正申請 {#term-correction-request}

スタッフが自身の勤怠記録に対して訂正を依頼する申請です。承認されるまで記録は確定扱いになりません。

## 申請中 {#term-pending-request}

修正申請が提出済みで、まだ完了していない状態です。管理者側の承認待ちを示します。

## 表示なし {#term-no-display}

勤怠判定ステータスが画面上で空表示になる状態です。当日、休日、利用開始日前、勤怠管理対象外など、判定対象外の日に使われます。内部的には `None` に対応します。

## 実労働時間 {#term-net-working-hours}

勤務開始から勤務終了までの時間から、休憩時間合計を差し引いた純粋な労働時間です。

## 振替休日 {#term-substitute-holiday}

特定日に勤務した代わりに、別の日を休日として取得する制度です。

## 特別休暇 {#term-special-leave}

有給休暇とは別に付与される特別な休暇です。就業規則に基づき利用条件が定義されます。
