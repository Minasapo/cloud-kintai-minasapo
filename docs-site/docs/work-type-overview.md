# 勤務形態の見方

このページは、アプリ内で設定する「勤務形態」の意味と、画面や運用への影響をまとめる共通ガイドです。

勤務形態はスタッフごとの設定項目です。選択肢は次の 2 つだけです。

- 平日勤務 (`weekday`)
- シフト勤務 (`shift`)

「勤務ステータス」はその時点の状態表示、「勤務形態」はスタッフに設定される働き方の種類です。名前が似ていますが、別の概念です。

## まず押さえる違い

| 項目 | 平日勤務 | シフト勤務 |
| --- | --- | --- |
| 想定する働き方 | 平日を中心に働くスタッフ | 曜日を固定せずシフトで働くスタッフ |
| 休日の基本的な見方 | 土日・祝日・会社休日は通常、勤務判定の対象外 | 曜日に関係なく勤務対象になりうる |
| シフト画面との関係 | スタッフ向けシフト画面の表示対象外 | スタッフ向けシフト画面の表示対象 |
| 迷ったときの確認先 | 管理者へ確認する | 管理者へ確認する |

## 画面上で影響を受けるポイント

### 勤怠一覧や確認画面

- 平日勤務は、土日・祝日・会社休日を通常は勤務判定の対象外として扱います。
- シフト勤務は、曜日に関係なく勤務日になりうるため、休日の扱いが平日勤務と異なります。
- そのため、同じ日付でも勤務形態によって「確認が必要かどうか」の見え方が変わることがあります。

勤務ステータスそのものの意味を確認したい場合は、[勤務ステータスの見方](./work-status-overview) を参照してください。

### シフト画面

- シフト勤務のスタッフは、シフト画面での確認や編集が日常運用に直結します。
- 平日勤務のスタッフには、スタッフ向けヘッダーのシフト項目は表示されません。
- 平日勤務のスタッフが直接アクセスしても、利用対象外メッセージを表示します。

スタッフ向けの操作手順は [シフトを確認・編集する](./staff/shift) を参照してください。

### スタッフ管理画面

- 管理者はスタッフ登録や編集時に勤務形態を設定します。
- 設定を変えると、その後の勤怠確認や休日の扱いに影響します。
- 迷った場合は、現場の運用に合わせて管理者が確認して設定します。

管理者向けの操作手順は [スタッフを管理する](./admin/staff-management) を参照してください。

## スタッフ向けの見方

- 自分の勤務形態は、日々の勤怠確認やシフト確認の前提になります。
- 休日なのに要確認になっている、または逆に表示されないと感じた場合は、まず勤務形態の違いを疑ってください。
- 自分で勤務形態を変更する画面は想定せず、変更が必要な場合は管理者へ相談します。

## 管理者向けの見方

- 勤務形態は、スタッフ登録時とスタッフ情報更新時に確認する基本項目です。
- 平日勤務かシフト勤務かを誤ると、勤怠確認時の休日判定やシフト運用の見え方がずれます。
- 設定変更時は、対象スタッフの実際の運用と合っているかを先に確認してください。

## よくある迷いどころ

### 土日に働く人は必ずシフト勤務ですか

必ずしもこのページだけでは判断しません。実際の運用ルールに合わせて管理者が判断します。

### 休日の見え方が想定と違います

勤務形態の違いで、同じ日付でも判定対象になるかどうかが変わることがあります。まず管理者に勤務形態設定を確認してください。

### 勤務ステータスとの違いが分かりません

- 勤務形態: スタッフに設定される属性
- 勤務ステータス: 今の状態や、その日の記録結果の表示

状態表示を確認したい場合は [勤務ステータスの見方](./work-status-overview) を参照してください。
