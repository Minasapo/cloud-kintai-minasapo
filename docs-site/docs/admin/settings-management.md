# 設定画面を管理する

このページは、管理者向けの設定画面全体をまとめた運用ガイドです。
起点は /admin/master/job_term で、/admin/master 配下の設定を対象にしています。

## 対象範囲

- 対象: /admin/master 配下の設定画面
- 読者: 運用管理者
- 更新基準: ルーティング定義と設定ナビゲーション定義

参照:
- src/router/adminChildRoutes.tsx
- src/features/admin/layout/model/adminSettingsNavigation.ts

## 基本

### 集計対象月

- 目的: 月ごとの締め日を定義し、月次運用の基準を作る
- 主要項目:
  - 締め日
  - 月情報
  - 一覧の追加/編集/削除
  - CSV一括登録
- 保存時の挙動:
  - 追加・編集・削除時に GraphQL mutation を実行
  - 実行後は一覧が更新される

### カレンダー設定

- 目的: 休日区分とイベント日を管理し、勤務判定の基準カレンダーを整える
- 主要項目:
  - 法定休日タブ
  - 会社休日タブ
  - イベントタブ
  - 日付・名称・有効/無効
  - 年月フィルター
- 保存時の挙動:
  - 各タブで追加・編集・削除時に GraphQL mutation を実行
  - 保存後は対象一覧に即時反映される

## 勤務ルール

### 勤務時間

- 目的: 標準の勤務時間帯と休憩時間帯を定義する
- 主要項目:
  - 始業時刻
  - 終業時刻
  - 休憩開始時刻
  - 休憩終了時刻
- 保存時の挙動:
  - 保存ボタンで AppConfig を更新
  - 更新後に設定再取得を実行

### 午前/午後休

- 目的: 半休運用の時刻と有効/無効を定義する
- 主要項目:
  - AM休暇開始/終了

  - 半休機能の有効/無効
- 保存時の挙動:

  - 更新後に設定再取得を実行

- 主要項目:

- 保存時の挙動:

### 特別休暇

## シフト・申請

## 運用上の注意

- 勤務ルール系の変更は、運用中の打刻・集計・表示へ影響します。月次運用の切り替えタイミングで変更してください。
- 設定変更後の表示反映は、画面や機能によって再取得タイミングが異なります。必要に応じて画面再読み込みを行ってください。
- 複数管理者が同時に同一設定を編集する場合、最後の保存内容が反映される前提で運用してください。
