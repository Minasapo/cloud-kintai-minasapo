# 設定画面を管理する

このページでは、管理者が設定画面で何を見直し、変更後にどこを確認すればよいかを説明します。

勤怠ルールと入力補助に関する設定は、`/admin/master` ではなく [勤怠一覧](./attendances) 右上の `設定` から見直します。
オフィスモードで使う `/office`、`/office/qr`、`/office/qr/register` は独立ロール向け画面ではなく、管理者設定に紐づく特殊運用画面として扱います。

設定画面で編集できる項目そのものを確認したい場合は、[設定項目一覧を確認する](./settings-item-list) を参照してください。

## この画面の目的

- 日々の運用に関わる設定を見直す
- 変更前に影響範囲を把握する
- 変更後に関連画面で反映を確認する

## 対象範囲

- 対象: 管理画面の設定メニュー
- 読者: 管理者
- 目的: どの設定を、どの場面で変更するかを判断する

## よく使う設定

| カテゴリ     | 画面           | 用途                                 |
| ------------ | -------------- | ------------------------------------ |
| 基本         | 集計対象月     | 締め日と集計期間の基準を決める       |
| 基本         | カレンダー設定 | 休日、イベント日を管理する           |
| 勤怠一覧設定 | 勤務時間       | 標準の始業、終業、休憩時間を設定する |
| 勤怠一覧設定 | 出勤モード     | 打刻運用の方式を切り替える           |
| データ連携   | 稼働統計       | スタッフ向け稼働統計ページの表示可否を切り替える |
| シフト、申請 | シフト         | シフト運用の前提を設定する           |
| シフト、申請 | ワークフロー   | 申請カテゴリを管理する               |
| データ連携   | 外部リンク     | 打刻画面などで使うリンクを管理する   |
| システム     | テーマ         | 画面の配色を設定する                 |

## 設定を変更するときの進め方

1. 変更対象の設定を決める
1. 影響を受ける画面を確認する
1. 変更内容を保存する
1. 関連画面を開いて反映を確認する

## 主要な設定の見方

### 集計対象月、カレンダー設定を見直す

- 月次締め前や新しい月の運用開始前に確認します。
- 集計対象月は、どの期間を基準に確認、集計するかに関わります。
- カレンダー設定は、休日やイベント日が想定どおりに扱われるかを左右します。
- 変更後は、勤怠一覧やダッシュボードで対象期間や表示内容に違和感がないか確認します。

### 勤務時間、出勤モードを見直す

- 打刻ルールを変更したいときや、勤務の標準時間を更新したいときに確認します。
- 勤務時間は、標準の始業、終業、休憩時間の基準になります。
- 出勤モードは、現場の打刻運用に合った方式になっているかを確認します。
- 変更後は、打刻画面や勤怠関連画面で、利用者が迷わず操作できる状態かを確認します。

### 稼働統計の表示を見直す

- スタッフに年間の稼働実績を見せるかどうかを切り替えたいときに確認します。
- 稼働統計を無効にすると、スタッフ向けメニューから非表示になり、直接アクセスもできなくなります。
- 変更後は、スタッフ画面のナビゲーションと遷移結果を確認します。

### シフト、ワークフローを見直す

- 申請やシフト運用の前提を変更したいときに確認します。
- シフト設定は、管理者側のシフト運用や対象者の見え方に影響します。
- ワークフロー設定は実装上は専用設定ダイアログを持ちますが、現行の一般公開ルートはありません。現在の公開運用では [申請を承認する](./request-approval) を基準にし、設定内容の確認は [ワークフロー設定を確認する](./workflow-settings) を参照してください。
- 変更後は、管理者画面の申請導線やシフト関連画面で意図どおりに扱えるかを確認します。

### 外部リンク、テーマを見直す

- 利用者に案内しているリンク先を更新したいときや、画面の見え方を調整したいときに確認します。
- 外部リンクは、打刻画面などから遷移する先が最新かどうかを見直します。
- テーマは、日常利用で見づらさがないかを確認するための設定です。
- 変更後は、実際の利用画面を開いて、リンク遷移や表示崩れがないことを確認します。

### オフィスモード、エクスポート、開発者設定を見直す

- オフィスモードは、QR 打刻用の特殊運用を有効化するときに確認します。
- エクスポートは、JSON 出力による保守作業が必要なときに確認します。
- 開発者設定は、通知機能など内部向けの補助設定を切り替えるときに確認します。
- いずれも日常の一般運用より、導入や保守、検証時に利用する前提です。

## 変更前に確認すること

- 何のために設定を変えるのかを明確にする
- 変更によって影響を受ける画面を先に洗い出す
- 月次締め直前や利用者が多い時間帯かどうかを確認する
- 変更後に確認する画面を決めてから保存する

## 変更後に確認すること

| 変更した設定   | 確認先の画面                                 | 確認すること                               |
| -------------- | -------------------------------------------- | ------------------------------------------ |
| 集計対象月     | [ダッシュボードを確認する](./dashboard)   | 表示対象期間や集計の基準に違和感がない     |
| カレンダー設定 | [勤怠一覧を確認する](./attendances)       | 休日や対象日の扱いが想定どおりになっている |
| 勤務時間       | [勤怠を管理する](./attendance-management) | 勤怠確認時の前提が変更内容とずれていない   |
| 出勤モード     | [ダッシュボードを確認する](./dashboard)   | 利用者導線や運用説明と矛盾がない           |
| 稼働統計       | [稼働統計を確認する](../staff/attendance-statistics) | メニュー表示と集計画面の導線が想定どおり   |
| シフト         | [シフトを管理する](./admin-shift)         | シフト運用の前提が意図どおり反映される     |
| ワークフロー   | [申請を承認する](./request-approval)      | 承認対象の扱いに違和感がない               |
| 外部リンク     | [ダッシュボードを確認する](./dashboard)   | 利用者向け導線から正しいリンクへ遷移する   |
| テーマ         | [ダッシュボードを確認する](./dashboard)   | 日常利用で見づらさや崩れがない             |

## よくある場面

### 月次締め前に見直す

1. 集計対象月を確認する
1. カレンダー設定を確認する
1. 勤怠一覧やダッシュボードで対象期間の見え方を確認する

### 打刻運用を調整する

1. 勤務時間と出勤モードを確認する
1. 変更前に影響を受ける利用者と画面を整理する
1. 変更後に打刻関連画面で操作の分かりやすさを確認する

### 稼働統計の公開範囲を調整する

1. 稼働統計設定を確認する
1. スタッフに表示するかどうかを切り替える
1. スタッフ画面でメニュー表示と遷移結果を確認する

### 申請運用を整える

1. ワークフローとシフトの設定を確認する
1. 承認や管理の流れに影響がないか確認する
1. 関連画面で申請やシフトの扱いを確認する

## 注意点

- 勤務ルールの変更は、当日の打刻や集計に影響する場合があります。
- 締め処理の直前は、必要最小限の変更にします。
- 変更後は、対象画面で表示を再確認します。
