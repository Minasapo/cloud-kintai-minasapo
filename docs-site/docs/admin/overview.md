# 管理者向けドキュメント

このセクションは、クラウド勤怠でスタッフの勤怠を管理する管理者向けです。

## 管理者の主な役割

管理者は、スタッフから提出された修正申請の承認・却下や、勤怠状況の確認・管理を行います。

主な業務は次のとおりです。

- 修正申請の承認フロー: スタッフから提出された打刻修正申請を確認し、承認または却下します。
- 勤怠状況の確認: スタッフごとの勤怠状況を一覧で確認し、要確認データに対応します。
- 月次締め前の確認: 対象期間に未処理の申請や要確認データが残っていないかを確認します。
