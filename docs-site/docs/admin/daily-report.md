# 日報を管理する

このページでは、管理者が /admin/daily-report 画面でスタッフの日報を確認し、未対応の日報を解消する手順を説明します。

## この画面の目的

- 日報の提出状況を一覧で把握する
- 未確認・提出済みなどのステータスを優先順で処理する
- コメントやリアクションでフィードバックを残す
- 必要な範囲の日報を CSV で出力して共有・保管する

## 主な確認項目

- ステータス集計: 提出済み、未確認、承認済みの件数
- 一覧フィルタ: ステータス、スタッフ、開始日、終了日
- 日報本文: タイトル、内容、作成日時、更新日時
- 反応履歴: 管理者コメント、リアクション

## 日次の運用手順

1. /admin/daily-report を開き、ステータス集計で未処理件数を確認する
1. フィルタ（ステータス、スタッフ、日付）で対象を絞り込む
1. 一覧から対象日報の詳細を開き、内容と更新日時を確認する
1. 必要に応じてコメントまたはリアクションを追加する
1. 対応後に一覧へ戻り、未処理件数が減っていることを確認する

## 主要な操作

### フィルタで対象を絞る

- ステータスで「未確認」「提出済み」などを切り替えて優先対応します。
- スタッフ名で担当者単位に絞り、確認漏れを防ぎます。
- 開始日と終了日で対象期間を限定し、月次確認にも使えます。

### 詳細を確認する

- 一覧の対象行から詳細画面を開けます。
- 詳細では日付、投稿者、タイトル、本文、更新時刻を確認できます。
- 必要な場合は分割表示で詳細パネルを開き、一覧と並べて確認します。

### コメントとリアクションを追加する

- コメントで確認内容や依頼事項を記録できます。
- リアクションで確認済みの意思表示を残せます。
- 追加した内容は履歴として残るため、対応状況の共有に利用できます。

### CSV を出力する

- 現在の絞り込み結果を CSV として出力できます。
- 期間別やスタッフ別のデータ共有が必要なときに利用します。

## 注意点

- まず未確認や提出済みを優先して処理し、承認済みは必要時のみ再確認してください。
- フィルタ条件が残っていると対象が欠けて見えるため、件数確認時は「すべて」に戻して再確認してください。
- コメント追加前に対象スタッフ・対象日付を再確認し、誤投稿を防いでください。
