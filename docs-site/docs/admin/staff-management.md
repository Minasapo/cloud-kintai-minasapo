# スタッフを管理する

このページでは、管理者が /admin/staff 画面でスタッフアカウントを管理する手順を説明します。

## この画面の目的

- スタッフ情報を検索し、必要なメンバーをすぐに見つける
- アカウント状態（有効・無効）とステータス（確認済み・パスワード変更必要）を把握する
- 新規スタッフ作成、ユーザー同期、アカウント操作を実施する

## 一覧で見るべき項目

- 登録スタッフ、有効アカウント、パスワード変更待ちの件数
- 名前、メールアドレス、権限、勤怠管理対象、勤務形態
- アカウント状態（有効・無効）
- ステータス（確認済み・パスワード変更必要）

## 日次の運用手順

1. /admin/staff を開き、件数カードで全体状況を確認する
1. 検索欄で対象スタッフを絞り込み、該当アカウントを特定する
1. 必要に応じて以下を実施する
   - スタッフ登録: 新規スタッフを作成する
   - ユーザー同期: Cognito とスタッフ情報の差分を同期する
   - アカウント操作: 有効化・無効化・削除を実行する
1. 操作後に一覧へ戻り、状態と件数が想定どおり反映されていることを確認する

## 主要な操作

### スタッフ登録

- 画面上部の「スタッフ登録」から新規作成できます。
- 作成時は、名前、メール、権限、勤務形態、勤怠管理対象などを設定します。

### ユーザー同期

- 「ユーザー同期」は、Cognito のユーザー情報とスタッフ一覧の差分を反映するための操作です。
- まれに同期漏れがある場合に実行してください。

### アカウント有効化・無効化

- 各行のメニューから実行できます。
- 無効化したアカウントはログインできなくなります。

### アカウント削除

- 各行のメニューから実行できます。
- 削除前に確認ダイアログが表示され、削除は取り消せません。

## 注意点

- パスワード変更待ちが多い場合は、対象スタッフに初回ログイン後の対応を案内してください。
- 有効・無効の切り替えは業務影響が大きいため、対象者を再確認してから実行してください。
- 削除は不可逆操作です。無効化で運用可能なケースは、まず無効化を優先してください。
