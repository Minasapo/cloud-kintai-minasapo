# ダッシュボードを確認する

このページでは、管理画面ダッシュボード（/admin）でその日の優先対応を素早く判断するための見方を説明します。

## 画面の目的

ダッシュボードは、管理者が最初に確認する全体把握の画面です。勤怠・日報・申請の状況を一か所で確認し、次に開くべき管理画面を決めます。

## 主な表示カード

### 勤務中スタッフ数

- 現在勤務しているスタッフ数を確認できます。
- 現場の稼働状況を把握し、必要に応じて勤怠一覧へ進みます。

### 日報ステータス

- 当日の日報の提出状況と承認状況を確認できます。
- 未確認がある場合は、日報管理へ進んで優先対応します。

### 保留中申請

- 保留中の勤怠修正申請とワークフロー申請を確認できます。
- 件数が増えている場合は、承認処理を優先します。

### 勤務状況チャート

- 期間内の勤務状況を可視化して確認できます。
- 偏りや急な変化がないかを見て、必要なら勤怠画面で詳細を確認します。

## ダッシュボードからの主な遷移先

| 確認したい内容           | 開く画面                                | 主な目的                 |
| ------------------------ | --------------------------------------- | ------------------------ |
| 要確認の勤怠を処理したい | [勤怠一覧を確認する](./attendances.md)  | 要確認データの解消       |
| 日報の未承認を解消したい | [日報を管理する](./daily-report.md)     | 提出済み日報の確認・承認 |
| 申請を処理したい         | [申請を承認する](./request-approval.md) | 承認・却下の実行         |

## 日次運用の使い方

1. 朝: ダッシュボードで勤務中スタッフ数と保留中申請を確認する
1. 日中: 日報ステータスを見て未処理を減らす
1. 締め前: 保留中申請と要確認が残っていないか再確認する

## 注意点

- 表示件数は運用状況に応じて更新されるため、処理前に最新状態を確認してください。
- 画面構成や表示項目は将来の改善で変更される場合があります。
