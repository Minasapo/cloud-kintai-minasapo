# ワークフロー設定を確認する

このページでは、ワークフロー設定ダイアログが持つ設定内容を整理します。

現状の一般公開 router では `/admin/workflow` が `NotFound` 扱いのため、通常運用でこの設定画面を開くことはできません。ここでは「現在の公開運用ではどう扱うか」と「実装として何を持っているか」を分けて説明します。

## 現在の公開運用

この節は、管理者が今すぐ使える画面の説明ではなく、現行の公開導線と実装上の設定機能を切り分けるための前提です。申請処理そのものは通常の管理導線で行い、ここでは「どんな設定が内部的に存在するか」を把握する目的で参照します。

- 日々の申請確認、承認、却下は [申請を承認する](./request-approval) を基準に進めます
- ワークフロー設定そのものは、一般利用者向けには公開導線がありません
- このドキュメントサイト上では、設定の存在と実装依存を把握するための参考情報として扱います

## 実装上持っている設定

実装に残っている設定は、主に申請フォームの見え方やカテゴリの扱いを制御するものです。どれも日常の承認操作とは別系統のため、運用ルールを変えたいときほど「申請者側にどう見えるか」まで意識して確認する必要があります。

### 申請カテゴリの表示順

- 申請カテゴリを並べ替える
- 並び順は一覧フィルタや新規作成画面の表示順に影響する
- `デフォルトに戻す` により標準順へ戻せる

### 申請カテゴリの有効 / 無効

- 各カテゴリを申請画面や一覧に表示するかを切り替える
- `欠勤` は別の AppConfig 条件にも依存する

### テンプレート管理

- `その他` 申請向けのテンプレートを作成、更新、削除する
- テンプレート名、タイトルテンプレート、詳細内容テンプレートを保持する

## 確認しておくべき影響先

設定値を直接触れない状態でも、影響先を把握しておくと将来 route を復活させるときや実装調査をするときに役立ちます。特にカテゴリ順序とテンプレートは、一覧と作成画面の両方にまたがって効く前提で見ておくのが安全です。

- スタッフ向けワークフロー新規作成画面
- ワークフロー一覧のカテゴリ表示
- `その他` 申請の初期入力内容
- 開発者向けの AppConfig 依存仕様

## 注意点

このページは公開運用の手順書ではなく、実装差分を誤解しないための補助資料です。現行 route と docs の説明がずれやすい領域なので、変更を入れる場合は利用者向けページより先に現行導線との整合を確認します。

- 現行の公開 route ではこの設定画面へ到達できません
- もし route を復活または再導入する場合は、`screen-list`、管理者ガイド、developer docs を同時に更新する必要があります
- 通知画面やメール文面には旧 `/admin/workflow/:id` 導線を前提にした記述が残っている可能性があります
