# 休憩時間申請をレビューする

このページでは、管理者が休憩時間関連の申請を判断する際の実務観点をまとめます。

## この画面の目的

- 休憩時間の修正申請を、締め処理に間に合うように判断する
- 法定休憩の目安を満たしているかを確認する
- 画面表示値と保存データの差を理解したうえで承認する

## 承認前に確認する項目

- 勤務開始・終了時刻に矛盾がないか
- 休憩時間の開始・終了が業務実態に合うか
- 休憩合計が法定休憩の目安を下回っていないか
- 修正理由が具体的か

## 法定休憩の目安

- 勤務が 6 時間を超える場合: 休憩 30 分以上を確認する
- 勤務が 8 時間を超える場合: 休憩 1 時間以上を確認する

## 自動補完を前提にした確認ポイント

- 退勤打刻の条件により、昼休憩が自動反映されることがある
- すでに休憩登録がある場合は自動追加されないため、手入力との差分を確認する
- 自動反映の有無だけで判断せず、勤務全体の整合性を優先する

## 表示値と実データの差を扱う

ダッシュボードの表示では、未記録の昼休憩帯を一時的に差し引いて見せることがあります。承認判断は、詳細画面の保存済み休憩データと申請理由を基準に行ってください。

## 差し戻し判断の目安

- 休憩時刻が重複し、実態を説明できない
- 休憩不足を補う意図が不明確で、理由欄も不足している
- 退勤時刻だけ修正され、休憩との整合が崩れている
