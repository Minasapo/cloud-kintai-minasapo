# ワークフロー管理画面の参考情報

このページは、実装コード上に残っている管理者向けワークフロー管理画面についての参考情報です。

現行の公開 router では `/admin/workflow` が一般公開ルートに含まれておらず、通常運用の手順としては利用しません。日常運用では [申請を承認する](./request-approval) を参照し、設定内容の確認は [ワークフロー設定を確認する](./workflow-settings) を参照してください。

## 現在の位置づけ

ここで把握したいのは、「実装が存在すること」と「今の公開運用で使うこと」は別だという点です。コード調査や通知導線の確認ではこの旧管理画面の存在が重要ですが、利用者向けガイドとしては現行の承認導線を優先して読む必要があります。

- `src/pages/admin/AdminWorkflow/**` に一覧、詳細、設定ダイアログの実装が残っています。
- `src/router/adminChildRoutes.tsx` では `workflow/*` が `NotFound` として扱われます。
- このドキュメントサイトでは、この画面を日常運用の正本にはしません。

## 現在参照すべきページ

- 承認や却下の現在運用: [申請を承認する](./request-approval)
- 設定一覧の棚卸し: [設定項目一覧を確認する](./settings-item-list)
- 旧管理画面が持っていた設定内容: [ワークフロー設定を確認する](./workflow-settings)
- 実装仕様: [ワークフロー機能仕様](../developer/workflow)

## 注意点

この差分を放置したまま docs や通知だけを更新すると、画面遷移の説明と実際の route がずれます。ワークフロー関連の変更は、実装、通知 URL、利用者向けガイドを同時に確認するものだと考えてください。

- 通知画面やメール文面には `/admin/workflow/:id` 相当の URL を前提にした記述が残っている場合があります。
- ルーティング変更時は、利用者向け docs だけでなく developer docs と通知導線の整合も合わせて確認してください。
