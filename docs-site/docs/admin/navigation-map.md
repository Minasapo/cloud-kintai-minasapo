# 画面遷移マップ（管理者向け）

このページは、管理者が目的ごとにどの画面へ進めばよいかをまとめた案内です。

## 最短導線

日次の全体確認
→ 管理画面ダッシュボード
→ 保留中申請と日報ステータスを確認
→ 優先度の高い画面へ移動

申請対応
→ 管理画面
→ 申請一覧
→ 詳細確認
→ 承認または却下

要確認の削減
→ 勤怠一覧
→ 要確認を抽出
→ 優先順で処理
→ 未処理件数を再確認

## 画面名ベースの導線表

| 目的                     | 開始画面               | 次に開く画面                                                                                     | 完了条件                           |
| ------------------------ | ---------------------- | ------------------------------------------------------------------------------------------------ | ---------------------------------- |
| その日の優先対応を決める | 管理画面ダッシュボード | [勤怠一覧](./attendances.md) / [日報管理](./daily-report.md) / [申請承認](./request-approval.md) | 優先対応する対象が明確になる       |
| 修正申請を承認する       | 管理画面               | 申請詳細画面                                                                                     | 承認処理が完了して未処理件数が減る |
| 修正申請を却下する       | 管理画面               | 申請詳細画面                                                                                     | 却下理由を記載して処理完了になる   |
| シフト計画を更新する     | シフト計画管理         | シフト計画編集画面                                                                               | 対象期間の計画が保存される         |
| 要確認を減らす           | 勤怠一覧               | 勤怠詳細または申請詳細                                                                           | 要確認件数が減少する               |
| 操作履歴を監査する       | 操作ログ               | 勤怠一覧 / 申請一覧                                                                              | 対象操作の経緯が確認できる         |
| 月次締め前の確認         | 勤怠一覧               | 申請一覧                                                                                         | 未処理申請が残っていない           |
| スタッフ情報を管理する   | スタッフ管理           | スタッフ編集画面                                                                                 | 対象アカウントの状態更新が完了する |

参照: [ダッシュボードを確認する](./dashboard.md)

## 申請を処理したい

1. 管理画面の申請導線を開く
1. 申請内容と理由を確認する
1. 承認または却下する

参照: [申請を承認する](./request-approval.md)

## シフト計画を更新したい

1. シフト計画管理画面を開く
1. 対象年を切り替えて計画対象期間を確認する
1. 月別の申請期間と日別人数を更新する
1. 保存時刻を確認して完了する

参照: [シフト計画を作成する](./shift-plan.md)

## 要確認データを減らしたい

1. 勤怠一覧で要確認を確認する
1. 優先度の高い申請から処理する
1. 未処理件数を再確認する

参照: [勤怠一覧を確認する](./attendances.md)

参照: [勤怠を管理する](./attendance-management.md)

## 操作履歴を確認したい

1. 操作ログ画面を開く
1. 期間と操作者または対象者で絞り込む
1. 詳細を開いて更新内容を確認する

参照: [操作ログを確認する](./operation-logs.md)

## 日報の未対応を減らしたい

1. 日報管理画面を開く
1. 未確認と提出済みを優先して絞り込む
1. 詳細を確認し、コメントやリアクションで対応内容を残す

参照: [日報を管理する](./daily-report.md)

## スタッフ情報を管理したい

1. スタッフ管理画面を開く
1. 検索で対象スタッフを特定する
1. 有効化・無効化・編集を実行する

参照: [スタッフを管理する](./staff-management.md)
