# 画面遷移マップ（管理者向け）

このページは、管理者が目的ごとにどの画面へ進めばよいかをまとめた案内です。

## 最短導線

日次の全体確認
→ 管理画面ダッシュボード
→ 保留中申請と日報ステータスを確認
→ 優先度の高い画面へ移動

申請対応
→ 管理画面
→ 申請一覧
→ 詳細確認
→ 承認または却下

要確認の削減
→ 勤怠一覧
→ 要確認を抽出
→ 優先順で処理
→ 未処理件数を再確認

## 画面名ベースの導線表

| 目的                     | 開始画面               | 次に開く画面                       | 完了条件                           |
| ------------------------ | ---------------------- | ---------------------------------- | ---------------------------------- |
| その日の優先対応を決める | 管理画面ダッシュボード | 勤怠一覧 / 日報管理 / ワークフロー | 優先対応する対象が明確になる       |
| 修正申請を承認する       | 管理画面               | 申請詳細画面                       | 承認処理が完了して未処理件数が減る |
| 修正申請を却下する       | 管理画面               | 申請詳細画面                       | 却下理由を記載して処理完了になる   |
| 要確認を減らす           | 勤怠一覧               | 勤怠詳細または申請詳細             | 要確認件数が減少する               |
| 月次締め前の確認         | 勤怠一覧               | 申請一覧                           | 未処理申請が残っていない           |

参照: [ダッシュボードを確認する](./dashboard.md)

## 申請を処理したい

1. 管理画面の申請導線を開く
1. 申請内容と理由を確認する
1. 承認または却下する

参照: [申請を承認する](./request-approval.md)

## 要確認データを減らしたい

1. 勤怠一覧で要確認を確認する
1. 優先度の高い申請から処理する
1. 未処理件数を再確認する

参照: [勤怠一覧を確認する](./attendances.md)

参照: [勤怠を管理する](./attendance-management.md)
