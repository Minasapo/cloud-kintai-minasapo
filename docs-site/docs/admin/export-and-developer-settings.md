# エクスポートと開発者設定を確認する

このページでは、日常運用よりも保守、検証、内部利用に近い設定をまとめます。

対象は次の 2 つです。

- エクスポート
- 開発者向け設定

## エクスポート

エクスポートは、設定や登録内容を外へ持ち出して確認するときの保守向け機能です。普段の管理作業で常用するものではないため、実行する理由と保存先を先に決めてから使う前提で運用します。

- パス: `/admin/master/export`
- 目的: 設定や登録データを JSON として出力する
- 権限: オーナー権限ユーザーのみ

### できること

- モデル単位の個別エクスポート
- 全対象モデルの一括エクスポート
- 一括実行時の進捗確認

### 運用時の注意

- 一括エクスポート中は画面遷移せず完了を待つ
- 出力ファイルには設定や登録データが含まれるため、保存先と共有範囲を管理する
- 保守や調査以外の通常運用では、常用機能として扱わない

## 開発者向け設定

開発者向け設定は、一般利用者向けの業務設定ではなく、検証や内部確認のために UI 表示を切り替える用途を想定しています。現場ルールを直接変えるものではない一方で、見える導線は変わるため、利用者権限ごとの差分確認が欠かせません。

- パス: `/admin/master/developer`
- 目的: 内部向けの補助設定を切り替える

### 現在確認できる設定

- 通知機能(開発中)
  - 有効にすると、ヘッダーの通知アイコンと通知一覧導線が表示される
  - 一般利用者向けの通常設定ではなく、検証や内部利用向けの扱い

### 運用時の注意

- 現場運用に関わる一般設定とは分けて扱う
- 変更後は通知アイコンや通知一覧への導線を実画面で確認する
- 管理者ユーザーとスタッフユーザーの両方で導線差分を確認する

## 変更後に確認すること

ここでの確認は、設定値が保存できたかよりも、保存結果が想定した画面差分として現れているかを見るためのものです。特に通知導線は権限や route 差分の影響を受けやすいため、片方の画面だけで判断しないようにします。

| 設定 | 確認先 | 観点 |
| --- | --- | --- |
| エクスポート | ダウンロードした JSON | 対象データが含まれているか、形式が崩れていないか |
| 通知機能 | ヘッダー、通知一覧、ワークフロー導線 | 導線が表示されるか、不要なユーザーに出ていないか |
