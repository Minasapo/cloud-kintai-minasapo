# 勤怠一覧を確認する

このページでは、勤怠一覧で優先度をつけて対応する手順を説明します。

一覧に表示される OK・要確認・申請中・表示なし の意味を共通整理で確認したい場合は、[勤務ステータスの見方](../work-status-overview.md) を参照してください。

## この画面の目的

- 当日の全体状況と未対応の有無を把握する
- 要確認の勤怠を抽出し、対応順を決める
- 申請対応が必要なデータを見つけて承認フローへつなげる

## 一覧で見るべき項目

- スタッフ名
- 勤務日
- 勤怠判定ステータス（OK、要確認、申請中、表示なし）
- 出勤・退勤時刻の欠落有無

## 日次の確認手順

1. 勤怠一覧を開き、当日の件数感を確認する
1. 「要確認」または「申請中」を優先して対象を抽出する
1. 勤怠内容を確認し、必要に応じて申請の承認・却下対応へ進む
1. 対応後に一覧へ戻り、未処理件数が減っていることを確認する

## 要確認データの扱い

- 打刻漏れや時刻矛盾は、対象日の勤怠詳細で事実確認を行う
- 申請中データは、修正理由と時刻整合性を確認して判断する
- 同一スタッフで複数日が該当する場合は、締め日が近い日付から優先する

### 休憩時間確認の観点

- 長時間勤務で法定休憩の目安を満たしているか
- 休憩修正申請の理由が具体的か
- 表示上の補正と保存データを取り違えていないか
